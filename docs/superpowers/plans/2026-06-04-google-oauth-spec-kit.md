# Google OAuth And Spec Kit Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Google OAuth entry points to the study room app, track linked auth providers in the database, and initialize GitHub Spec Kit in the current project.

**Architecture:** The web app keeps email OTP login and adds a Google OAuth redirect login path. OAuth callbacks are handled in-app with `exchangeCodeForSession`, and provider metadata is mirrored into a user-owned `user_identities` table through RLS-safe upserts.

**Tech Stack:** Vite, React, TypeScript, Supabase Auth, Supabase SQL migrations, GitHub Spec Kit.
